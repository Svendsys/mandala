# Baumhard Projection Layer Review — scene_builder / tree_builder / connection / scene_cache / border / portal_geometry

## Architecture assessment

The "two pipelines" CODE_CONVENTIONS §3 describes are further along consolidation than the docs admit — and the leftover half is now mostly dead weight rather than a live second path. Today every visual role reaches the GPU through `Tree<GfxElement, GfxMutator>`: node text/chrome via the mindmap tree, borders via `border_node_data` → border tree, portals via `portal_pair_data` → portal tree, and connections/labels/handles/section-frames via `RenderScene` elements *re-projected into per-role trees* by `scene_rebuild.rs`. The flat renderer consumers (`rebuild_border_buffers`, `rebuild_edge_handle_buffers`, `rebuild_connection_label_buffers`, `fit_camera_to_scene`) have zero call sites, yet `build_scene_with_cache` still manufactures `TextElement`s (full text clones), `BorderElement`s (full style + palette resolution), `PortalElement`s (full style + layout), and a resolved background color on every scene build — all discarded in production — while borders and portals are then resolved a second time for their tree builders. So the dual-pipeline cost today is not divergent rendering (the shared helpers `resolve_border_style`, `border_run_specs`, `build_border_regions`, `resolve_portal_endpoint_style`, and the zoom-window resolvers genuinely prevent style drift); it is (a) per-frame duplicate/dead computation in the hot path, (b) one real cross-pipeline correctness drift (`BorderRunSpec.line_height_pt` honored only by the dead flat path), and (c) misleading documentation that keeps the dead half looking alive. The remaining migration is small and well-defined: delete the dead flat consumers, stop emitting the dead scene fields, move the three remaining courier roles' emission next to their tree builders, and `RenderScene` collapses into "connection sampling + cache + selection overlay IR." Quality is otherwise high — the mutator-vs-rebuild parity tests, cache invariance tests, and Unicode coverage are exemplary — but §B1/§B7 discipline slips in this layer: O(N²) model walks per drag frame, an O(samples×nodes) clip filter, per-lookup String keys in the metric cache, and the per-frame border/portal double-resolve all land on the mobile budget.

**Concrete consolidation order** (each step independently shippable):
1. **Delete the dead flat consumers** (zero risk): `Renderer::rebuild_border_buffers` / `rebuild_edge_handle_buffers` / `rebuild_connection_label_buffers` (scene_buffers.rs:26-161), the perpetually-empty `border_buffers`/`edge_handle_buffers`/`connection_label_buffers` fields + their draw-chain hooks (renderer/mod.rs:358-372, render.rs:321-333), and `fit_camera_to_scene` (hit.rs:33-55).
2. **Stop dead emission**: split `build_node_elements` into an AABB/clip pass (all the connection pass needs) and drop `TextElement`/`BorderElement`/`PortalElement`/`background_color` from `build_scene_with_cache`. Port the affected scene tests to the tree outputs (portal tests → `portal_pair_data`; text/border tests → the mindmap/border trees). This deletes the border/portal double-resolve for free.
3. **Move the couriers home**: `SectionFrameElement`, `ConnectionLabelElement`, and handle emission move next to their tree builders (they need nothing from `RenderScene` except the shared model). `RenderScene` shrinks to `connection_elements` + `edge_handles`.
4. **Final seam**: fold the cache-aware connection sampling into `tree_builder::connection` (the `SceneConnectionCache` survives unchanged; only the element struct moves), then delete `RenderScene` and update CODE_CONVENTIONS §3 / CONCEPTS §3.
Blockers per step are only tests + the node_pass AABB coupling; no interactive feature depends on the flat outputs.

## Quest digests not covered by numbered findings

- **Quest 4 (cache)**: invalidation is caller-managed and broadly correct today — `scene_cache.clear()` at ~20 sites (undo, console exec, color commit, text edit, resize release at event_mouse_click.rs:1076/1131, animation tick, camera-fit geometry_dirty), per-edge `invalidate_edge` in the edge-handle drain (throttled_interaction/edge_handle.rs:101), zoom via `ensure_zoom`, deletion via `retain_keys`. The stale-fast-path hazard is even pinned by `test_cache_fast_path_serves_stale_when_model_moved_without_offsets` (tests/cache.rs:519). Residual risks are findings 10 (duplicate keys) and 12 (config-blind fast path). The cache is consulted only by the connection body pass; label pass, edge-handle pass, `hit_test_edge`, and `edge_label_drag` each rebuild paths independently (bounded counts except the hit test — finding 21).
- **Quest 5 (path math)**: `point_at_t` / `tangent_at_t` / `normal_at_t` correct with documented degenerate fallbacks; `closest_point_on_path` Newton refinement has a seed-fallback divergence guard and is verified against finite differences and inflection cases (connection/tests.rs:744-967) — correct and allocation-free; quadratic→cubic promotion is done once in `build_connection_path` and every consumer (labels, handles, hit test, drag) goes through it — consistent. Costs: finding 14 (arc-table allocation).
- **Quest 9 (zoom gating)**: consistent across both pipelines. Node containers/sections/borders inherit `node.zoom_window()` (tree_builder/node.rs:117,160; border.rs BorderNodeData); connections use `edge.zoom_window()` in both scene element and tree area; labels `label_zoom_window`; portals `portal_endpoint_zoom_window` (replace-not-intersect pinned by tests in both pipelines). Deliberate exceptions, all defensible: section frames use `ZoomVisibility::default()` with a written justification (tree_builder/section_frame.rs:128-134); selection handles (edge/resize) carry no window — selection chrome renders even when the selected edge itself is zoom-hidden, a minor UX oddity, not a bug. `TextElement` has no zoom field but is production-dead (finding 2). No role silently drops an authored window.

### 4. BorderRunSpec.line_height_pt is dropped by both live border tree paths — the "gappy rails" fix only exists on the dead pipeline
Severity: P1 | Category: correctness | Confidence: high (drift is certain; visual severity med — affects patterns whose glyph ink-height ≠ font_size)
Files: /home/user/mandala/lib/baumhard/src/mindmap/border.rs:436-458,524-564; /home/user/mandala/lib/baumhard/src/mindmap/tree_builder/border.rs:246-258,323-356; /home/user/mandala/lib/baumhard/src/mindmap/tree_builder/section_frame.rs:135-152; /home/user/mandala/src/application/renderer/scene_buffers.rs:66-74; /home/user/mandala/src/application/renderer/tree_walker.rs:137-163

### 6. Connection clip filter is O(samples × nodes) per frame with no spatial pruning
Severity: P2 | Category: performance | Confidence: high
Files: /home/user/mandala/lib/baumhard/src/mindmap/scene_builder/connection.rs:63-101,422-434
Evidence: `emit_connection_element` runs `point_inside_any_node(p, node_aabbs)` — a linear scan over every visible node's AABB — for every sampled glyph position of every visible edge, every frame (all three cache paths rerun it by design, connection.rs:16-19). 300 edges × ~50 samples × 250 nodes ≈ 3.75M AABB tests per frame, plus a fresh `Vec<(f32,f32)>` collect per edge per frame (line 82-86).

Effort: M
